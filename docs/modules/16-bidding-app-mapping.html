Gaps & Recommended Actions

  1. Stone-level valuation breakdown — add the Lot Stone entity (module 03) and the /lots/{id}/stones endpoint (module 04). Decide: does the full breakdown persist, or only the final bid/ct?
  2. Media — add stone photo/video upload (/stones/{id}/media) and storage. Confirm size/format limits and whether media is mandatory for sign-off.
  3. Grading masters — publish Shape / Color / Clarity master lists (module 14) so the app validates against the same codes.
  4. Lot number vs description — app should capture them separately rather than one combined string, so the ERP maps cleanly to rough-lot numbering later.
  5. Currency — confirm the app's "$" is the tender currency (USD default) so bid totals reconcile.
  6. Pieces semantics — a stone row may represent N similar pieces; confirm this maps to a sub-lot/parcel rather than N separate records.
Outcome: once these are agreed, modules 03–04 become the frozen /api/v1 contract the app builds against.
