Creating Masters via the MeghaOS MCP

Runbook for creating Donda's master records by driving the MeghaOS MCP. The what (fields per master) is locked; the how (exact MCP tool calls) is provisional until the MCP is live.

Overview & Status

⚠ Provisional — MeghaOS MCP not yet deployed to production. The call convention and tool names below are placeholders to be confirmed by introspecting the real MCP once it's connected. The field/payload data is final — sourced live from the master registry (mockups/megha-erp/masters-data.js), so it always matches the mockups.

The Donda ERP is built on the MeghaOS platform; masters are created through its MCP rather than hand-built screens. This page documents, per master: the entity key, every field (with its payload key), a sample record, the order to create them in, and the open questions for the MCP team.

Worked example — create a Supplier (generic form)

// PLACEHOLDER call — confirm tool name, auth & response against the live MCP meghaos.master.create({ "entity": "supplier", "data": { "name": "Bonas Couzyn (Antwerp) NV", "display_name": "Bonas Couzyn", "country": "Belgium", "type": "Expo / Tender", "payment_terms": "NET5", // FK → payment-terms master "currency": "USD", // FK → currency master "status": "Active" } }) → expected: { "id": "<assigned-by-meghaos>", "entity": "supplier", "status": "created" }
Conventions used here: payload keys are the column label lowercased with non-alphanumerics → _ (e.g. "Company Name" → company_name, "Rate → USD" → rate_usd). FK-style fields reference other masters by their code/name — create the referenced master first (see Creation Order).

Creation Order (dependencies)

Create independent masters first, then the ones that reference them, so foreign keys resolve.

TIER 1 — no dependencies (create first) currency · country · payment-terms · expense-type · hs-code stone-type · shape · color · clarity · lot-numbering TIER 2 — reference Tier 1 mine → country group-company → currency transport → country TIER 3 — reference Tiers 1–2 supplier → payment-terms, currency customer → currency (+ won-lot stones use shape/color/clarity)
Idempotency: re-running creation should not duplicate. Until the MCP's behaviour is known, key each master by its natural code (or name) and check-before-create.

Open Items (confirm once MCP is live)

  1. Tool surface — generic create(entity, data) vs per-entity create_<entity>? Exact tool name(s)?
  2. Auth — how the MCP authenticates (token/company scope) and which Donda company context masters are created under.
  3. Required vs optional — which fields MeghaOS enforces; default values; server-side validation rules.
  4. IDs & references — does MeghaOS return an ID? Are FK fields passed by code, name, or ID?
  5. Idempotency / upsert — is there an upsert, or must we check-before-create by code?
  6. Bulk create — can we create many rows in one call (e.g. all currencies/grading codes)?
  7. Entity keys — confirm MeghaOS's entity identifiers map to our keys (supplier, mine, currency, …).
