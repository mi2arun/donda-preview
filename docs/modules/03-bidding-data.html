Lot Stones — Valuation Breakdown FROM APP

In the lot-capture screen each lot is broken into stone estimates — the appraiser's view of how the rough will polish, which justifies the bid. As the ERP stores full expo data, this breakdown persists here. See the lot-capture mockup and the field mapping.

Lot Stone / Sub-Estimate DETAIL — per lot

FieldTypeNotes
Stone / Sub IDAUTOSystem reference
Lot Ref *LookupParent expo lot

Rough Weight (ct) *Decimal(2)Rough carats for this estimate
Polish Yield %DecimalExpected polished yield

PiecesIntegerGroup of N similar stones (default 1)
Polish WeightCALCRough × Yield %
Estimated ValueCALCPolish Weight × Price/ct
MediaAttachmentsPhotos / videos captured per stone
Bid math: stone estimates roll up to the lot — R-PRICE = Σ value ÷ Σ rough, bid floor = R-PRICE × (1 − discount%), bid total = bid/ct × Σ rough. Only the chosen bid $/ct is the committed decision; the rest is decision support.
