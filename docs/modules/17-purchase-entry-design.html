Purchase Entry — Build Design

The first module being built: Won Lot → Purchase Invoice (BOM of rough sub-lots) → Rough Lot Master. This is the locked, build-ready design. See the interactive purchase-invoice mockup.

Model & Hierarchy

PURCHASE INVOICE = ONE winning lot (parent rough lot 1-26) │ └─ BOM line items = SUB-LOTS / stones (1-26-A, 1-26-B, 1-26-C …) each = one rough-diamond item → one sub-lot stock unit
  • One purchase invoice = one won lot. A supplier invoice with several parcels (e.g. #20260645 → 070/085/086) is one lot whose parcels are sub-lots 1-26-A/B/C.
  • Inventory lives at the sub-lot level — each 1-26-A is the stock unit (own carats, value, balance, status). Parent lot 1-26 is a roll-up grouping.
  • Expenses allocated by value (proportional to each sub-lot's goods amount).
  • One sub-lot per parcel line, pieces kept together; split further at sale.
  • Tagging: header tags the won lot; each line tags the won lot's stone. On confirm → won lot = Invoiced.
  • Third-party rough (no won lot): lot & sub-lot numbers entered manually; no tag.
Supersedes: the v1 functional spec modelled each invoice parcel as a separate top-level lot (1-26, 2-26, 3-26). The build adopts one lot per invoice with sub-lots (1-26-A/B/C). Modules 07 / 08 describe the field-level detail.

Entry Flow

  1. Create Purchase Invoice → choose Buying Company + Supplier, currency & exchange rate, supplier invoice no/date, received date.
  2. Pull from Won Lots → pick the won lot (filtered to that supplier/company, status Received). Its stones load as BOM sub-lot lines, pre-filled but editable against the real supplier invoice.
  3. System assigns the lot number 1-26 (group sequence) and sub-lot suffixes A/B/C…; per-line mining origin.
  4. Add expenses (freight, duty, insurance…) → allocated by value.
  5. Confirm → creates parent Rough Lot 1-26 (roll-up) + sub-lot stock units 1-26-A/B/C, flips won lot to Invoiced, and (later) posts accounting.

Numbering

  • Lot No = {seq}-{yy} from a group-shared sequence (unique across all companies). Manual override for third-party.
  • Sub-lot = parent + letter suffix -A, -B… auto-assigned per lot, one per BOM line.
  • Reservation timing (assign on Draft-create vs. on Confirm) is an open item — affects gaps in the sequence.

Open Items (parked)

  1. Carat variance — won/bid carats vs. actually-invoiced carats: received figure wins; show the delta as a flag.
  2. Lot-number reservation timing — on Draft-create (risk of gaps if cancelled) vs. on Confirm (no gaps, but number unknown while drafting).
  3. Multi-level split — can a sub-lot 1-26-A be split again at sale (1-26-A-1)? Carried from the sales module.
  4. Rough grading at receipt — do we capture shape/color/clarity on the purchase line, or only carats + description?
